おーい先輩、ちょっと聞いてもいいですか？
お、どうした。

先輩、CXLメモリプーリングの話を聞いたんですけど、実際に買える製品ってあるんですか？
あるある。Samsung、SK hynix、Micronの主要メモリメーカーがもう製品を出してる。CXL Type-3メモリデバイスってやつだ。

Type-3って何ですか？
CXL（Compute Express Link、シーエックスエル）にはType-1、Type-2、Type-3の3種類がある。Type-1はキャッシュ機能付きアクセラレータ、Type-2はメモリ内蔵アクセラレータ。で、Type-3はメモリバッファ専用だ。PCIeスロットに挿すDRAMみたいなもんだと思っていい。

具体的な製品を教えてください。
SamsungのCMM-D（CXL Memory Module DRAM-based、シーエムエムディー）は128GB〜512GBのラインアップでPCIe Gen5 x8接続。特徴はメモリプーリング機能で、複数ホストで共有できる。SK hynixのNGD（Next Generation DRAM、エヌジーディー）はHBM技術を応用して最大1TB、ECC内蔵で信頼性が高い。MicronのCZ120（シージー120）は最大256GBで、独自プロセスにより消費電力を競合比15%削減してる。

レイテンシはどのくらいですか？
DRAM比で2〜5倍。とはいえNVMe SSDの数十マイクロ秒と比べれば圧倒的に速い。DRAMとストレージの間を埋める新しいメモリ階層って位置づけだ。

Linuxで使うには何が必要ですか？
カーネル5.12以降でCXLの基本サポートが入ってる。管理ツールはdaxctl（ダックスクティーエル）とndctl（エヌディークティーエル）。daxctlでDAXデバイスとして公開すればアプリが直接メモリを操作できるし、kmem（ケイメム）ドライバを使えば通常のシステムメモリとして認識させることもできる。その場合CXLメモリは新しいNUMAノードとして追加される。

NUMAってやつですね。
そう。CXLメモリをNUMAエクステンダとして使うのが最も実用的な使い方。ローカルDRAMより遅いことを前提に、ホットデータはDRAM、コールドデータはCXLメモリに置く階層化構成が効果的だ。LinuxのAuto NUMA Balancing（オートヌーマバランシング）を使えば、アクセス頻度に応じて自動的にページを昇格させることもできる。

データセンターの構成が変わりそうですね。
CXL対応CPUが本格普及すれば、サーバのメモリ容量を柔軟に拡張できる標準構成になるだろう。LLMの推論やインメモリDBの需要が増える中で、この立ち位置はますます重要になる。
