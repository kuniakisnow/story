データセンターにおけるメモリ容量の需要は、大規模言語モデル（LLM、エルエルエム）の推論やインメモリデータベースの拡大に伴って急増している。従来のDRAM（ディーラム）は容量拡張に物理的な制約があり、コスト面でも限界が見え始めている。この課題に対するソリューションとして注目されるのが、CXL（Compute Express Link、シーエックスエル）Type-3メモリデバイスである。

## CXL Type-3とは何か

CXLはPCIe（ピーシーアイエクスプレス）の物理層をベースに、CPUとアクセラレータ、メモリバッファ、ストレージデバイスの間で一貫性のあるメモリアクセスを提供するインターフェース規格である。Type-1（キャッシュ機能を持つアクセラレータ）、Type-2（メモリ内蔵アクセラレータ）に対して、Type-3はメモリバッファ専用デバイスとして位置づけられる。

Type-3デバイスが実装するプロトコルはCXL.memのみである。これはホストCPUからデバイス上のメモリ領域へのロードストアアクセスを可能にするプロトコルであり、CXL.io（コンフィギュレーションやI/O用）は併用されるがCXL.cache（キャッシュコヒーレンシ用）は実装しない。言い換えればType-3は、ホストから見てPCIeスロットに挿すDRAMのような存在である。ホットプラグ対応やメモリの論理分割といった機能も規格として準備されている。

## 実製品の比較

CXL Type-3メモリデバイスは2023〜2024年にかけて主要メモリメーカーから製品化が相次ぎ、2026年現在では第二世代の製品も登場している。

Samsung（サムスン）のCMM-D（CXL Memory Module DRAM-based、シーエムエムディー）は、DDR5（ディーディーアールファイブ）DRAMをベースにCXLインターフェースを統合した製品である。容量は128GBから512GBまでラインナップされ、PCIe Gen5 x8インターフェースを備える。特徴的なのはメモリプーリング機能であり、複数ホストで同一のCMM-Dを共有できる点である。これによりメモリ利用効率を大幅に向上させることができる。

SK hynix（エスケーハイニックス）のNGD（Next Generation DRAM、エヌジーディー）は、同社のHBM（High Bandwidth Memory、エイチビーエム）技術を応用した製品である。NGDは1TBまでの大容量を実現し、電力効率でCMM-Dを上回る。またECC（イーシーシー）機能を内蔵し、長時間運用における信頼性を高めている。

Micron（マイクロン）のCZ120（シージー120）は、Micron初のCXL Type-3製品であり、最大256GBの容量を提供する。他社製品との差別化ポイントは、消費電力の最適化とコストパフォーマンスである。Micron独自の1ベータプロセスDRAMを採用することで、競合製品と比較して消費電力を約15%削減している。

これらの製品はいずれもレイテンシにおいてDRAM比で2〜5倍程度の値を示す。純粋なDRAMよりは遅いが、NVMe（エヌブイエムイー）SSDの数十マイクロ秒と比較すると圧倒的に高速であり、メモリ容量拡張とストレージアクセス回避の中間的なポジションを占める。

## ホストソフトウェア対応

CXL Type-3デバイスを活用するには、ホスト側のオペレーティングシステムとアプリケーションの対応が不可欠である。Linuxカーネルでは5.12以降でCXLの基本サポートがマージされ、6.0以降でType-3デバイスのホットプラグやメモリゾーン管理が実用レベルに達している。

具体的な管理ツールとして、daxctl（ダックスクティーエル）とndctl（エヌディークティーエル）が重要な役割を果たす。daxctlはCXLメモリ領域をDAX（Direct Access、ダイレクトアクセス）デバイスとして公開し、アプリケーションがファイルシステムを経由せずに直接メモリを操作するためのデバイスノードを作成する。一方ndctlはCXLメモリの容量分割や名前空間管理を担う。

またLinuxカーネルのkmem（ケイメム）ドライバを使用すると、CXLメモリを通常のシステムメモリとして認識させることができる。この場合、CXLメモリは新たなNUMA（Non-Uniform Memory Access、ヌーマ）ノードとして追加され、カーネルのページアロケータが自動的に使用する。ただしCXLメモリとローカルDRAMではレイテンシに差があるため、numactl（ヌーマクティーエル）を用いた明示的なポリシー設定が推奨される。

## NUMAエクステンダとしての活用

CXL Type-3メモリデバイスの最も実用的な使い方は、NUMAエクステンダとしての導入である。従来のNUMA構成ではCPUソケットに直結するDRAM容量に上限があったが、CXL Type-3を追加することでメモリ容量を数テラバイト規模に拡張できる。

NUMAエクステンダとして使う際の運用上のポイントは、CXLメモリへのアクセスレイテンシがローカルDRAMよりも高いことを前提としたメモリ配置設計である。例えば頻繁にアクセスするホットデータはローカルDRAMに配置し、アクセス頻度が低いコールドデータや巨大なモデルパラメータはCXLメモリに配置する階層化メモリ構成が効果的である。Linuxでは数回のアクセスでページをCXLメモリからローカルDRAMに昇格させるAuto NUMA Balancing（オートヌーマバランシング）機能も利用可能である。

## まとめ

CXL Type-3メモリデバイスは、DRAMの容量制約とNVMe SSDのレイテンシ制約の谷間を埋める新しいメモリ階層を実現する。Samsung CMM-D、SK hynix NGD、Micron CZ120といった実製品がすでに出揃い、Linuxのdaxctlやkmemによるホスト対応も整いつつある。NUMAエクステンダとして導入することで、既存サーバのメモリ容量を柔軟に拡張できる。CXL対応CPUの本格普及に伴い、今後のデータセンターアーキテクチャにおける標準的な構成要素になる可能性が高い。
